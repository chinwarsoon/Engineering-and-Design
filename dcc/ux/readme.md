# Section 1. Requirements
1. `User Experience (UX)` should be considered in developing `Engineering and Design System (DES)`.
2.  Refer to ISO 9241-210:2019 — Ergonomics of human-system interaction, Part 210: Human-centred design for interactive systems. `User Experience` is a person's perceptions and responses that result from the use or anticipated use of a product, 
system or service.
3. Four principles of human-centred design:
- Design based on explicit understanding of users, tasks, and environments.
- Users involved throughout design and development.
- Design driven and refined by user-centred evaluation.
- The process is iterative.
4. Law of User Experience:
- Hick's Law: More choices will result slower decisions. Reduce the number of options on screen at any one time. Prioritise one or two primary actions, demote the rest. Group related options into categories (pick category, then item). Use progressive disclosure for advanced features. A homepage with one clear CTA outperforms one with seven equal-weight links.
- Von Restorff Effect: What stands out is remembered
- Serial Position Effect: First and last items remembered best. Put the most important option first or last in a list. Place the primary CTA at the top or bottom of the page, never in the middle. In navigation: home goes first, account/help goes last. In onboarding: lead with the strongest example, end with the 
strongest action. Anything you bury in the middle, the user will forget.
- Miller's Law: Limited working memory requires chunking data. Working memory holds about four chunks of data.
- Tesler's Law: Push complexity into the system design
- Peak-End Rule: Peaks and endings shape memory
- Fitts's Law: Large, near targets will win
- Doherty Threshold: Fast feedback sustains engagement
- Gestalt Principles: Brains will group visual information
- Jakob's Law: Users expect familiar patterns
- Aesthetic-Usability Effect: Beautiful feels easier to use
- Postel's Law: Consider flexible inputs and structured output
